### 5 mai 2026 — Implémentation algorithme des transferts

Implémentation complète en 8 tâches (subagent-driven development) :

**`tools/transfers.py`** (module pur, sans I/O) :
- `estimate_theta(sim, percentile=75)` — seuil automatique depuis la distribution de similarité
- `run_transfers(slugs, sim, theta, max_k, force_assign, dry_run, initial_partition, max_iter=100)` → partition complète
- Algo 1 : partition initiale (ordre aléatoire, centroïdes incrémentaux) — O(k × N)
- Algo 2 : amélioration Gauss-Seidel jusqu'à max_iter ou aucun transfert > min_gain_delta
- `force_assign` : réaffecte les pages "poubelle" (label=-1) au centroïde le plus proche
- `dry_run` : compte les transferts sans les appliquer (mesure de qualité)
- `initial_partition` : mode incrémental — Algo 1 sur les nouvelles pages uniquement

**`tools/cluster.py`** mis à jour : branche `algo="transfers"`, CLI `--theta`, `--max-k`, `--force-assign`, `--incremental`, `--dry-run`

**`tools/server.py`** mis à jour : `POST /cluster` avec les nouveaux paramètres, `GET /cluster-quality`

Premier run réel : `clustering-embeddings-transfers-0.404`, param=50, ~84 clusters, theta=0.404.

**Implémentation en 3 tâches** :

**`tools/kb_query.py`** — requête de proximité thématique :
```python
def kb_query(vec: np.ndarray, kb_dir: Path, top_k: int = 3) -> list[dict]:
    # Retourne les top_k axes les plus proches avec id, title, score, tags
```
Produit une entrée injectée dans le prompt LLM à l'ingestion :
```
Ce document est proche de :
- Philosophie du langage (0.81)
- IA et cognition (0.74)
```

**`tools/kb_update.py`** — mise à jour de la base depuis un wiki archivé :
- Charge les `cluster-*.md` d'un clustering archivé
- Calcule le centroïde + cohésion de chaque cluster via ses embeddings
- Critères d'exclusion : `status=garbage`, `size < min_size`, `cohesion < theta/2`
- Fusion avec un axe existant si similarité ≥ 0.85 (moyenne pondérée), sinon création d'un nouvel axe
- Écriture atomique (`.tmp` + `rename`) pour tous les fichiers

**`tools/kb_tags.py`** — dictionnaire de tags normalisés :
- Collecte tous les tags du wiki, les embed via BGE-M3
- Clustering cosinus (seuil 0.90) → forme canonique = tag le plus fréquent
- Exclut les hapaxes par défaut (`--keep-hapax` pour les conserver)
- Produit `tags_dict.json` + `tags_embeddings.npy`

## Architecture du pipeline complet

```
Signets Brave → bookmarks_to_raw.py → raw/
                                         ↓
raw/ → ingest.py (DeepSeek/Ollama) → wiki/sources/ (src-*.md)
                                         ↓
wiki/sources/ → embed.py (BGE-M3) → embeddings/embeddings.npy
                                         ↓
embeddings/ → cluster.py + transfers.py → wiki/clusterings/clustering-*/
                                         ↓
wiki archivé → kb_update.py → knowledge_base/axes/ + embeddings/ (centroïdes)
                                         ↓
knowledge_base/ → kb_query.py → contexte thématique injecté dans prompt ingest.py
```

Consultation : `query.py` (BM25 + BGE-M3 hybrid), `server.py` (Flask :5051 pour Obsidian)
