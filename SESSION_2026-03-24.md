## Lecture approfondie du code (résultats)

### expression_extractor.py

- Endpoint : `POST http://127.0.0.1:8989/v1/chat/completions`
- Chunking : `SemanticChunker` depuis `secretarius_local/vendor/chunk_data.py`
- Traitement par chunk, déduplication globale
- Filtrage verbatim : `expr in text` (simple `in` Python)
- Retour : `{"chunks": [...], "by_chunk": [...], "expressions": [...], "request_fingerprint": "sha256:..."}`

### semantic_graph.py

- 1 embedding par expression, 1 row Milvus par expression
- Champs Milvus : `id`, `vector` (1024 dim), `payload_json`, `doc_id`, `expression_id`, `expression_norm`, `keywords`, `type_note`
- Recherche actuelle : vecteur unique → top_k hits directs
- `semantic_graph_search_milvus()` accepte déjà une **liste** de vecteurs (`data=normalized`)
- Upsert intelligent : supprime les vieux rows du même `doc_id` avant insertion

### mcp_server.py

6 outils exposés : `ask_oracle`, `extract_expressions`, `semantic_graph_search`, `index_text`, `search_text`, `update_text`

### embeddings.py

- Modèle : BAAI/bge-m3
- Dimension : 1024
- L2-normalisation par défaut

### chef_orchestre.py

- Cycle ReAct : détection commande directe → appel LLM → parsing JSON → validation → appel outil
- Max 2 appels d'outils par cycle
- Recovery JSON : ferme les accolades/crochets manquants
- Bloque `ask_oracle` si non demandé explicitement

### document_pipeline.py

- Analyse : hashtags, date, doc_id, type_note, URLs, titre/incipit
- État d'indexation : `queued → extracting → embedding → upserting → done/error`
- Structure document : `secretarius.document.v0.1`

### Tests existants (ce qui est déjà couvert)

- Filtrage par score min, upsert/suppression Milvus, IDs stables
- Filtres keywords Milvus (AND/OR/NOT)
- Parsing doc_id, type_note, query keyword operators
- Cycle ReAct : JSON recovery, blocage oracle, markdown JSON
- 15 fichiers de tests, markers non encore déclarés dans pytest.ini

### test_model.ipynb (lora_local)

Montre le pipeline complet :
1. Chunking sémantique (SentenceTransformer `paraphrase-multilingual-MiniLM-L12-v2`)
2. `POST http://sanroque:8989/v1/chat/completions` avec prompt extracteur
3. `filtrer_expressions(texte, expressions)` — filtre verbatim avec normalisation unicode
